# Glossary

Terms and definitions used in the Claude Skills Collection.

## A

### Agent
An autonomous entity that can perform tasks. In the context of multi-agent orchestration, agents work together to complete complex tasks.

### Agent Orchestration
The process of coordinating multiple agents to work together on a task.

### Anthropic
The company behind Claude and Claude Code.

### Auto Mode
A mode where the collection automatically selects the most relevant skill for a task.

## C

### Claude
Anthropic's AI assistant.

### Claude Code
Anthropic's CLI tool for AI-assisted coding.

### Claude Skills Collection
This project - a comprehensive library of skills for Claude Code.

### CodeQL
A semantic code analysis platform developed by GitHub.

### CI/CD
Continuous Integration and Continuous Deployment.

### Commit Message
A message that describes the changes made in a git commit.

### Contributor
Someone who contributes to the project.

## D

### DevOps
Development Operations - practices that combine software development and IT operations.

### Docker
A platform for developing, shipping, and running applications in containers.

### Dependency
A package or library that a project depends on.

### Deployment
The process of releasing software to a production environment.

## E

### Environment
The context in which software runs, including operating system, dependencies, and configuration.

### External Integration
Integration with external services or APIs.

## F

### Feature
A new capability or functionality.

### Fork
A copy of a repository that allows independent development.

### Fuzzing
A software testing technique that provides random data as input to find bugs.

### Frontend
The user-facing part of a software application.

## G

### GitHub
A web-based platform for version control and collaboration.

### GitLab
A web-based DevOps platform.

### Git
A distributed version control system.

### GraphQL
A query language for APIs.

## H

### Helm
A package manager for Kubernetes.

### Hook
A mechanism for executing custom code at specific points in a workflow.

## I

### Issue
A report of a bug, feature request, or other item that needs attention.

### Integration
The process of combining different software components.

### Infrastructure
The hardware and software components that support an application.

## J

### JSON
JavaScript Object Notation - a lightweight data interchange format.

### JWT
JSON Web Token - a compact, URL-safe means of representing claims to be transferred between two parties.

## K

### Kubernetes
An open-source platform for managing containerized workloads and services.

### Key
A value used in encryption or authentication.

## L

### Linting
The process of analyzing code for potential errors, bugs, and style issues.

### License
A legal document that specifies how software can be used and distributed.

### Log
A record of events that occur in a system.

### LLM
Large Language Model - a type of AI model trained on vast amounts of text data.

## M

### MCP
Model Context Protocol - a protocol for integrating external tools with AI models.

### Multi-Agent Orchestration
The process of coordinating multiple agents to work together on a task.

### Merge
The process of combining changes from different branches.

### Module
A self-contained unit of code.

## N

### NestJS
A framework for building efficient, scalable Node.js server-side applications.

### Next.js
A React framework for building web applications.

### Node.js
A JavaScript runtime built on Chrome's V8 JavaScript engine.

### NPM
Node Package Manager - the default package manager for Node.js.

## O

### Open Source
Software that is freely available to use, modify, and distribute.

### ORM
Object-Relational Mapping - a technique for converting data between incompatible systems.

### OTP
One-Time Password - a password that is valid for only one login session or transaction.

## P

### PR
Pull Request - a request to merge changes into a repository.

### Pipeline
A set of automated processes that allow developers and DevOps professionals to work together.

### Plugin
A software component that adds a specific feature to an existing software application.

### Python
A high-level, interpreted programming language.

### Prisma
An ORM for Node.js and TypeScript.

## Q

### Query
A request for information from a database or API.

### Quality Assurance
The process of ensuring that software meets specified requirements.

## R

### Repository
A storage location for software packages.

### Release
A version of software that is made available to users.

### Review
The process of examining code or other artifacts for quality and correctness.

### Rollback
The process of reverting to a previous version of software.

## S

### Skill
A self-contained module that performs a specific task.

### Security
The practice of protecting systems, networks, and programs from digital attacks.

### Semgrep
A tool for static analysis that supports custom rules.

### Supply Chain
The network of all the people, organizations, resources, activities, and technology involved in creating and selling a product.

### Static Analysis
The analysis of software without executing it.

## T

### Testing
The process of evaluating and verifying that a software application works as expected.

### Terraform
An infrastructure as code tool that allows you to build, change, and version infrastructure.

### TypeScript
A superset of JavaScript that adds static typing.

### Test-Driven Development
A software development process that relies on the repetition of a very short development cycle.

## U

### UI/UX
User Interface / User Experience.

### Update
The process of bringing software up to date with the latest version.

### User
A person who uses software.

## V

### Version
A specific state of software at a particular point in time.

### Vulnerability
A weakness in a software system that could be exploited by a threat actor.

### Validation
The process of checking that data meets specified requirements.

## W

### Workflow
A sequence of processes that accomplish a specific task.

### Webhook
A method for an application to provide real-time information to another application.

### Worktree
A separate working directory linked to a Git repository.

## Y

### YAML
A human-readable data serialization language.

## Z

### Zod
A TypeScript-first schema validation library.

### Zone
A logical grouping of resources.
