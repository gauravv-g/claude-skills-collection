# Security Policy

## Supported Versions

This is a curated collection of skills. Security support depends on the source repositories.

## Reporting Security Vulnerabilities

If you discover a security vulnerability in any skill:

1. **Check the source repository** — Each skill comes from a specific source repository (see README.md)
2. **Report to the source repository** — Security vulnerabilities should be reported to the maintainers of the source repository
3. **Optional: Report here** — You can also open an issue here to track the vulnerability

### Security Best Practices

When using skills from this collection:

- Review skills before using them in production
- Keep your Claude Code installation updated
- Be cautious with skills that execute code or access external systems
- Never share sensitive information in skill configurations

### Skill Safety

This collection includes skills from various sources. While we curate the collection:

- We do not guarantee the security of individual skills
- Skills may execute code or access external systems
- Always review and understand what a skill does before using it

### Responsible Disclosure

If you find a security issue:

1. Do not create a public issue
2. Report it privately to the source repository maintainers
3. Allow time for the issue to be fixed before disclosing
